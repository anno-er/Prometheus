# 使用 `link` 和 `@import` 引入 CSS 样式表的区别

在 HTML 文档中，可以通过 `<link>` 标签或 CSS 中的 @import 语句来引用外部样式表。两者在使用方式、加载顺序、兼容性等方面有一些重要的区别。

## 对比维度详解

| 对比维度           | `<link>` 标签                                                 | `@import` 规则                                                            |
| ------------------ | ------------------------------------------------------------- | ------------------------------------------------------------------------- |
| **引入位置**       | 在 **HTML 文件的 `<head>` 中** 使用                           | 在 **CSS 文件内部** 使用                                                  |
| **加载时机**       | **并行加载**，不阻塞页面渲染（特别是使用 `preload` 等优化后） | **延迟加载**，会阻塞后续 CSS 解析，可能拖慢渲染                           |
| **加载顺序**       | 按照 HTML 中 `<link>` 出现的顺序 **同步加载 & 应用**          | 按照 `@import` 在 CSS 中出现的顺序 **依次加载**，**可能导致样式延迟生效** |
| **性能影响**       | 更优，支持并行加载、预加载等优化手段                          | 较差，会导致额外的 HTTP 请求延迟，阻塞渲染树构建                          |
| **浏览器兼容性**   | 所有浏览器都支持                                              | 所有现代浏览器支持，但老版本 IE 对 `@import` 的某些用法支持有问题         |
| **是否可动态加载** | 可通过 JavaScript 动态插入 `<link>` 实现动态加载              | 无法直接通过 JS 动态导入（必须通过 CSS 加载）                             |

## 加载方式

| 方式      | 引入场景      | 并行加载                            | 阻塞渲染                  | 性能评价 |
| --------- | ------------- | ----------------------------------- | ------------------------- | -------- |
| `<link>`  | HTML 直接引入 | ✅ 并行（多个 link 可同时下载）     | ❌ 阻塞渲染（需等 CSSOM） | 更优     |
| `@import` | CSS 内部引入  | ❌ 串行（一个 import 一个一个加载） | ✅ 阻塞后续 CSS 解析      | 较差     |
