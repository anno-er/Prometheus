# HTML 的 src 与 herf 有什么区别

HTML 中的 src 和 href 虽然都用于引用外部资源，但它们在核心功能和使用场景上有本质区别。

| 特性对比       | src (Source)                                          | href (Hypertext Reference)         |
| -------------- | ----------------------------------------------------- | ---------------------------------- |
| **核心功能**   | **嵌入**资源到当前文档                                | **建立链接/关联**到其他资源        |
| **语义含义**   | 表示资源的**来源地址**                                | 表示超文本的**引用目标**           |
| **常见标签**   | `<img>`, `<script>`, `<iframe>`, `<video>`, `<audio>` | `<a>`, `<link>`, `<base>`          |
| **加载行为**   | **阻塞渲染**：浏览器会暂停解析以加载和执行资源        | **非阻塞渲染**：资源在后台异步加载 |
| **与文档关系** | 资源是当前文档的**组成部分**                          | 资源与当前文档是**关联关系**       |

## 从加载行为看性能优化：

理解两者的加载行为对网页性能优化很重要。由于 src 会阻塞页面渲染，通常建议把引用了外部 JavaScript 文件的 script 标签放在文档底部，或者为其添加 async 或 defer 属性来改善加载体验。而 href 引用的资源（如通过 `<link>` 引入的 CSS 文件）通常是异步加载的，不会阻塞页面的解析和显示。

## 一个元素的特殊情况：

`<link>` 元素的情况比较特殊。当它用于引入外部样式表时，使用的是 href 属性（如 `<link rel="stylesheet" href="style.css">`），这是因为 CSS 被视为与文档关联但非嵌入的资源。这印证了 href 用于建立“关联”的核心功能。

## src 加载机制

浏览器在解析到带有 Src 属性的元素时,会立即向指定 URL 发送请求

### 阻塞情况:

阻塞行为:对于`<script>`,如果没有加 async 或 defer 属性,浏览器会暂停 HTML 的解析,等待脚本本加载并执行完成。
非阻塞行为:对于`<img>`和`<iframe>`资源加载是异步的,不会阻塞页面解析。

## href 加载机制

资源引用:浏览器解析到 href 时,标记该资源为外部引用,不会立即加载到页面中。

延迟加载:对于`<a>`标签,用户点击链接后,才会触发页面跳转。

预加载情况:对于`<link rel="stylesheet">` 浏览器会解析 CSS 并应用到页面中,可能会延迟页面的渲染(RendeBlocking)

## 优化行为

### src 的性能优化

**异步加载脚本**

1. 使用 async 属性让脚本与 HTML 解析并行加载,加载完成后立即热行
2. 使用 defer 属性让脚本与 HTML 解析并行加载,但执行顺序与页面解析顺序一致,且在解析完成后才执行。

**图片懒加载**

1. 添加 loading="lazy"属性,延迟加载不在视囗范围内的图片。

**CDN 优化**

1. 将常用的脚本(如 jQuery、FontAwesome)存储在 CDN 上,提高加载返康庭

**资源压缩:**

1. 对图像资源使用更高效的格式(如 WebP)。
2. 对脚本资源进行压缩和丑化(Minification)。

### href 的性能优化

**预加载关键资源**

使用`<link rel="preload" href="style.css" as="style">`指定关键资源,提高页面加载速度

**防止阻塞渲染**

将非关键 CSS 文件标记为延迟加载,例如:`<link rel="stylesheet"href=="style.css" media="print" onload="this.media="all';">`

**动态加载**

使用 JavaScript 动态创建 <1ink>标签以加载非必要资源。
