# 什么是 Node.js 的事件循环机制?它是怎么实现的?

## 什么是事件循环机制

Node.js 的事件循环（Event Loop）是 Node.js 处理非阻塞 I/O 操作的核心机制。它允许 Node.js 在单线程环境中执行大量异步操作，避免了传统多线程模型中的上下文切换开销。

事件循环本质上是一个无限循环，负责监听事件并将其分发给相应的事件处理器。它基于 JavaScript 的单线程特性，通过异步回调机制实现并发操作。

## 事件循环的实现原理

Node.js 采用单线程模型执行 JavaScript 代码，但其强大的并发处理能力源于事件驱动架构和非阻塞 I/O 操作。

- 非阻塞 I/O：当 Node.js 执行如文件读写或网络请求等 I/O 操作时，它不会停滞等待操作完成。相反，它会继续执行后续代码，待 I/O 操作完成后，通过回调函数处理结果。这就像餐厅厨师在炖汤时可以去准备其他菜品，汤好了再回来处理。

- 事件循环的协调作用：事件循环作为“调度中心”，持续监视调用栈（Call Stack）和事件队列（Event Queue）。一旦调用栈为空（即当前同步任务执行完毕），事件循环便按特定规则从事件队列中取出回调函数送入调用栈执行。这个过程循环往复，使 Node.js 能够处理大量异步任务。

## JS Event Loop

JavaScript 的事件轮询（Event Loop）是其实现异步编程的核心机制，它让单线程的 JavaScript 能够高效处理非阻塞的 I/O 操作。
事件轮询机制依赖于以下几个核心部分的协同工作：

- 调用栈（Call Stack）
  这是 JavaScript 引擎用来跟踪函数执行的一个简单数据结构。当你调用一个函数时，它会被推入（push）栈顶；当函数执行完毕返回时，它会被弹出（pop）栈。它负责执行所有的同步代码。

- 任务队列（Task Queue） 异步操作完成后，其对应的回调函数会在这里排队等待执行。任务队列又分为两种：

  - 宏任务队列（Macrotask Queue）：包含 setTimeout, setInterval, I/O 操作（如文件读写、网络请求）、UI 交互事件、setImmediate（Node.js 环境）等。
  - 微任务队列（Microtask Queue）：包含 Promise.then/catch/finally、MutationObserver（浏览器环境）、process.nextTick（Node.js 环境，且优先级高于其他微任务）等。

- 事件循环（Event Loop）
  它就像一位不知疲倦的协调员，其工作逻辑非常简单：持续检查调用栈是否为空。一旦调用栈空了，它就按照 “清空所有微任务 → 执行一个宏任务 → 再次清空所有微任务” 的顺序循环工作。这个过程确保了微任务总能优先于下一个宏任务得到执行。
