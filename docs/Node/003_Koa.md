# Koa 框架面试题

### 1. 什么是 Koa? 它有什么特点?

**定义：**
Koa 是由 Express 原班人马开发的轻量级 Node.js Web 框架，专注于现代中间件架构，旨在成为更小、更富有表现力、更健壮的 Web 框架。

**特点：**

- **轻量级**：核心代码非常精简，不捆绑任何中间件
- **现代语法**：基于 ES6+ async/await，告别回调地狱
- **洋葱模型**：独特的中间件执行机制
- **上下文封装**：将 request 和 response 封装到 ctx 对象
- **错误处理**：优雅的 try-catch 错误捕获机制
- **高度可定制**：按需选择中间件，灵活组合

---

### 2. Koa 和 Express 有什么区别?

| 特性         | Koa                              | Express                         |
| ------------ | -------------------------------- | ------------------------------- |
| 中间件模型   | 洋葱模型（支持等待响应）         | 线性模型（顺序执行）            |
| 异步处理     | 原生支持 async/await             | 基于回调，需要额外处理          |
| 内置功能     | 极简，无内置中间件               | 内置路由、静态文件等            |
| Context 对象 | 统一的 ctx 对象                  | 分离的 req/res 对象             |
| 错误处理     | try-catch 统一捕获               | 需要错误处理中间件              |
| 体积         | 更小（约 1KB）                   | 较大                           |
| 学习曲线     | 需要自己组合中间件               | 开箱即用                        |
| 社区生态     | 相对较小                         | 成熟丰富                        |

---

### 3. 什么是 Koa 的洋葱模型? 它是如何工作的?

**定义：**
洋葱模型是 Koa 中间件的核心机制，中间件像洋葱一样一层层包裹，请求从外向内穿过所有中间件，响应从内向外返回。

**工作原理：**

```
请求 → 中间件1 → 中间件2 → 中间件3 → 响应处理 → 中间件3 → 中间件2 → 中间件1 → 响应
         ↓           ↓           ↓              ↑           ↑           ↑
       前置逻辑    前置逻辑    前置逻辑       后置逻辑    后置逻辑    后置逻辑
```

**代码示例：**

```javascript
const Koa = require("koa");
const app = new Koa();

app.use(async (ctx, next) => {
  console.log(">> 中间件1 开始");
  await next();
  console.log("<< 中间件1 结束");
});

app.use(async (ctx, next) => {
  console.log(">> 中间件2 开始");
  await next();
  console.log("<< 中间件2 结束");
});

app.use(async (ctx, next) => {
  console.log(">> 中间件3 开始");
  ctx.body = "Hello Koa";
  console.log("<< 中间件3 结束");
});

// 执行顺序：
// >> 中间件1 开始
// >> 中间件2 开始
// >> 中间件3 开始
// << 中间件3 结束
// << 中间件2 结束
// << 中间件1 结束
```

**关键点：**

- `await next()` 之前的代码在请求阶段执行
- `await next()` 之后的代码在响应阶段执行
- 如果忘记 `await next()`，后续中间件将不会执行

### 4. Koa 中的 Context (ctx) 对象是什么?

**定义：**
Context 对象是 Koa 的核心概念，它将 Node.js 的 request 和 response 对象封装到一个对象中，提供了更简洁的 API。

**常用属性和方法：**

| 属性/方法              | 说明                         |
| ---------------------- | ---------------------------- |
| `ctx.request`          | 请求对象                     |
| `ctx.response`         | 响应对象                     |
| `ctx.method`           | 请求方法                     |
| `ctx.url` / `ctx.path` | 请求 URL/路径                |
| `ctx.query`            | 查询字符串对象               |
| `ctx.body`             | 响应体（获取或设置）         |
| `ctx.status`           | 响应状态码                   |
| `ctx.type`             | 响应 Content-Type            |
| `ctx.header`           | 请求头对象                   |
| `ctx.cookies`          | Cookie 操作对象              |
| `ctx.state`            | 跨中间件共享数据的命名空间   |
| `ctx.throw()`          | 抛出 HTTP 错误               |
| `ctx.redirect()`       | 重定向                       |

**示例：**

```javascript
app.use(async (ctx) => {
  ctx.status = 200;
  ctx.type = "application/json";
  ctx.body = {
    method: ctx.method,
    url: ctx.url,
    query: ctx.query,
  };
});
```

### 6. Koa 中 app.use 方法有什么作用?

**作用：**
`app.use()` 用于注册中间件函数，中间件会按照注册顺序依次执行。

**特点：**

- 接受一个 async 函数作为参数
- 中间件函数接收 ctx 和 next 两个参数
- 支持链式调用

**示例：**

```javascript
app
  .use(async (ctx, next) => {
    console.log("日志中间件");
    await next();
  })
  .use(async (ctx, next) => {
    console.log("认证中间件");
    await next();
  })
  .use(async (ctx) => {
    ctx.body = "Hello";
  });
```

### 7. 如何在 Koa 中实现路由?

**方式一：使用 koa-router（推荐）**

```javascript
const Koa = require("koa");
const Router = require("koa-router");

const app = new Koa();
const router = new Router();

router.get("/", (ctx) => {
  ctx.body = "首页";
});

router.get("/users/:id", (ctx) => {
  ctx.body = `用户ID: ${ctx.params.id}`;
});

router.post("/users", (ctx) => {
  ctx.body = "创建用户";
});

app.use(router.routes()).use(router.allowedMethods());
```

**方式二：手动实现简单路由**

```javascript
app.use(async (ctx) => {
  if (ctx.path === "/" && ctx.method === "GET") {
    ctx.body = "首页";
  } else if (ctx.path === "/about" && ctx.method === "GET") {
    ctx.body = "关于页面";
  } else {
    ctx.status = 404;
    ctx.body = "Not Found";
  }
});
```

### 8. Koa 如何处理错误?

**方式一：try-catch 捕获**

```javascript
app.use(async (ctx, next) => {
  try {
    await next();
  } catch (err) {
    ctx.status = err.status || 500;
    ctx.body = {
      message: err.message,
    };
  }
});
```

**方式二：ctx.throw() 抛出错误**

```javascript
app.use(async (ctx) => {
  if (!ctx.query.token) {
    ctx.throw(401, "未授权访问");
  }
  ctx.body = "成功";
});
```

**方式三：全局错误监听**

```javascript
app.on("error", (err, ctx) => {
  console.error("服务器错误:", err);
});

// 触发错误
app.use(async (ctx) => {
  ctx.throw(500, "服务器内部错误");
});
```

### 12. 什么是 app.context? 有什么作用?

### 16. Koa 和 Egg.js 有什么关系?

**关系：**
Egg.js 是基于 Koa 开发的企业级 Node.js 框架，由阿里团队维护。

**对比：**

| 特性       | Koa                    | Egg.js                       |
| ---------- | ---------------------- | ---------------------------- |

| 约定      | 无约定，自由组合       | 约定优于配置                 |

| 适用场景   | 小型项目、API 服务     | 大型企业应用                 |

### 17. 如何实现 JWT 认证中间件?

```javascript
const Koa = require("koa");
const jwt = require("jsonwebtoken");
const app = new Koa();

const secret = "your-secret-key";

const authMiddleware = async (ctx, next) => {
  const token = ctx.header.authorization?.replace("Bearer ", "");

  if (!token) {
    ctx.throw(401, "未提供认证令牌");
  }

  try {
    const decoded = jwt.verify(token, secret);
    ctx.state.user = decoded;
    await next();
  } catch (err) {
    ctx.throw(401, "无效的认证令牌");
  }
};

app.use(authMiddleware);

app.use(async (ctx) => {
  ctx.body = { user: ctx.state.user };
});
```

### 19. 为什么 Koa 没有内置路由?

**原因：**

1. **保持核心轻量**：Koa 设计理念是极简核心
2. **灵活选择**：开发者可以自由选择路由方案
3. **职责单一**：框架专注于中间件机制
4. **可扩展性**：便于社区开发各种路由实现

### 20. 如何优化 Koa 应用性能?

**优化策略：**

1. **启用集群模式**

```javascript
const cluster = require("cluster");
const os = require("os");

if (cluster.isMaster) {
  for (let i = 0; i < os.cpus().length; i++) {
    cluster.fork();
  }
} else {
  require("./app");
}
```

2. **使用缓存**

```javascript
const cache = new Map();

app.use(async (ctx, next) => {
  const key = ctx.url;
  if (cache.has(key)) {
    ctx.body = cache.get(key);
    return;
  }
  await next();
  cache.set(key, ctx.body);
});
```

3. **压缩响应**

```javascript
const compress = require("koa-compress");
app.use(compress());
```

4. **合理使用中间件顺序**

```javascript
app.use(logger); // 日志放最前
app.use(compress()); // 压缩放前面
app.use(static()); // 静态资源
app.use(router()); // 路由放最后
```

### 21. Koa 中间件如果忘记 await next() 会怎样?

**后果：**

- 后续中间件不会执行
- 响应可能提前结束
- 洋葱模型被破坏

**示例：**

```javascript
app.use(async (ctx, next) => {
  console.log("中间件1");
  next(); // 忘记 await
  console.log("中间件1 结束"); // 这行会在中间件2之前执行
});

app.use(async (ctx, next) => {
  console.log("中间件2");
  await next();
});

// 输出顺序错误：
// 中间件1
// 中间件1 结束
// 中间件2
```

### 25. Koa 的优缺点是什么?

**优点：**

- 核心极简，体积小
- 现代 async/await 语法
- 洋葱模型灵活强大
- 错误处理优雅
- 高度可定制

**缺点：**

- 需要自己组合中间件
- 社区生态不如 Express 成熟
- 学习成本相对较高
- 中间件选择多样，容易混乱

### 26. 如何实现一个简单的 koa-compose?

```javascript
function compose(middleware) {
  return function (ctx, next) {
    let index = -1;

    function dispatch(i) {
      if (i <= index) return Promise.reject(new Error("next() called multiple times"));
      index = i;
      let fn = middleware[i];
      if (i === middleware.length) fn = next;
      if (!fn) return Promise.resolve();
      try {
        return Promise.resolve(fn(ctx, dispatch.bind(null, i + 1)));
      } catch (err) {
        return Promise.reject(err);
      }
    }

    return dispatch(0);
  };
}
```

### 30. Koa 中 this 关键字在中间件中指向什么?

**答案：**

在 Koa 中间件中，`this` 的指向取决于函数类型：

- **箭头函数**：`this` 继承自外层作用域
- **普通函数**：`this` 指向当前 Context 对象

**推荐使用箭头函数**，通过参数 `ctx` 访问上下文：

```javascript
app.use(async (ctx, next) => {
  ctx.body = "Hello";
});
```
