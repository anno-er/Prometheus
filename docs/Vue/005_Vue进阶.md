# Vue 进阶

## DIFF 算法的原理

DIFF 算法的核心原理是通过比较新旧两棵虚拟 DOM 树,找出不同点,并且只更新必要的部分以达到高效更新真实 DOM 的目的。这种差异化更新策略能够显著提升性能,避免不必要的 DOM 操作。

## Vue SEO

Vue 是一个单页应用框架,默认情况下,搜索引擎爬虫无法直接获取到 Vue 应用的内容。为了优化 Vue 应用的 SEO,我们可以采取以下措施:

1. 服务器端渲染(SSR):使用 Vue Server-Side Rendering(Vue SSR)或者 Nuxtjs 框架,将 Vue 组件在服务器端渲染成 HTML,通过这种方式搜索引擎爬虫能够很容易地读取静态内容
2. 预渲染(Prerendering):使用 Prerender SPA Plugin 或者其他预道染工具在构建时生成 HTML 文件,然后部署这些文件到服务器。
3. Meta 标签:确保页面包括关键的 meta 标签,比如 title、description、keywords 等,使搜索引擎能够很好地理解页面内容。
4. 友好的 URL:使用 VueRouter 来创建用户和搜索引擎都能理解的友好 URL
5. 优化内容:确保内容相关性高,使用 H1 标签、高质量的图片、即良好的内容排版等

## 响应式 Object.defineProperty & Proxy

1. Object.defineProperty:
   - Object.defineProperty 是 JavaScript 中用于定义对象属性的方法。
   - 它可以用来定义对象的属性,包括属性值、是否可枚举、是否可配置等。
   - 响应式实现原理：
     - Vue 利用 Object.defineProperty 来实现响应式。
     - 当我们定义一个响应式数据时,会使用 Object.defineProperty 来为该数据定义一个 getter 和 setter。
     ```js
     Object.defineProperty(obj, "property", {
       get: function () {
         // 读取属性时触发
       },
       set: function (newValue) {
         // 设置属性时触发
       },
     });
     ```

2. Proxy:
   - Proxy 是 ES6 中新增的特性,用于创建一个对象的代理。
   - 代理对象可以拦截对目标对象的操作,例如读取属性、设置属性、调用方法等。
   - 通过 Proxy,我们可以在不修改目标对象的情况下,对其进行拦截和处理。
   - 响应式实现原理：
     - Vue 利用 Proxy 来实现响应式。
     - 当我们定义一个响应式数据时,会使用 Proxy 来创建一个代理对象。
     ```js
     const proxy = new Proxy(obj, {
       get: function (target, property) {
         // 读取属性时触发
       },
       set: function (target, property, newValue) {
         // 设置属性时触发
       },
     });
     ```

## 为什么在Vue3中很多api只能在setup中执行

setup函数是Composition API的唯一入口，它在组件实例创建之前、props解析之后同步执行。此时，组件实例尚未创建，因此this为undefined，无法访问data、computed、methods等选项。Vue通过一个内部的currentInstance变量来追踪正在初始化的组件实例，所有生命周期钩子与响应式API都需要注册到这个实例上。如果在setup外部调用这些API，currentInstance为空，将导致注册失败或响应式系统无法正确绑定到组件。

1. 响应式数据创建API：ref、reactive
2. 响应式数据读取API：watch、watchEffect、computed
3. 生命周期钩子：onBeforeMount、onMounted、onUpdated等
4. 组件实例访问API：getCurrentInstance、provide、inject
5. 模板引用与组件暴露：ref（用于DOM/组件引用）、defineExpose（在`<script setup>`中）
6. Props与Emit相关API：defineProps、defineEmits（在`<script setup>`中）
